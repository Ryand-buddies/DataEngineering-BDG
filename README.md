# Data Engineering Evaluation Project

Este proyecto está diseñado para evaluar habilidades en ingeniería de datos, cubriendo tres áreas principales: **SQL**, **PySpark** y **Python**. Contiene ejercicios prácticos que simulan problemas reales de ingeniería de datos.

## 🎯 Objetivos del Proyecto

- Evaluar conocimientos de SQL para consultas complejas y manipulación de datos
- Probar habilidades en PySpark para procesamiento de big data
- Verificar competencias en Python para ETL, análisis y manipulación de datos
- Simular escenarios reales de ingeniería de datos

## 🚀 Configuración Inicial

### Prerequisitos
- Python 3.8 o superior
- Java 8 o superior (para PySpark)
- Git

### Instalación

1. Clona este repositorio:
```bash
git clone https://github.com/Ryand-buddies/DataEngineering-BDG.git
cd DataEngineering-BDG
```

2. Instala las dependencias:
```bash
pip install -r requirements.txt
```

3. Ejecuta el script de configuración:
```bash
chmod +x setup/install.sh
./setup/install.sh
```

## 📚 Áreas de Evaluación

### 🗄️ SQL
- Consultas complejas con JOINs múltiples
- Funciones de ventana (window functions)
- CTEs (Common Table Expressions)
- Agregaciones avanzadas
- Optimización de consultas

### ⚡ PySpark
- DataFrames y RDDs
- Transformaciones y acciones
- Joins distribuidos
- Optimización de performance
- Manejo de datos grandes

### 🐍 Python
- Manipulación de datos con Pandas
- ETL pipelines
- Limpieza y transformación de datos
- Algoritmos de procesamiento
- Análisis estadístico básico

## 📊 Datasets Incluidos

Los datasets en la carpeta `data/` simulan datos de:
- Transacciones de e-commerce
- Logs de aplicaciones web
- Datos de sensores IoT
- Información de usuarios y productos

## 🎓 Cómo Usar Este Proyecto

1. **Para Candidatos**: Sigue las instrucciones en cada carpeta de ejercicios
2. **Para Evaluadores**: Revisa los criterios de evaluación en cada sección
3. **Para Aprendizaje**: Usa los ejercicios como práctica personal

## 📝 Criterios de Evaluación

- **Correctitud**: ¿La solución produce los resultados esperados?
- **Eficiencia**: ¿El código es optimizado para el rendimiento?
- **Legibilidad**: ¿El código es claro y bien documentado?
- **Buenas Prácticas**: ¿Se siguen las mejores prácticas del lenguaje/framework?
